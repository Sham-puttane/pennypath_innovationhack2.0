# PennyPath

**The Financial Order of Operations — for Everyone Else.**

PennyPath is a financial wellness app that builds personalized, mathematically defensible financial plans for underserved populations — gig workers, immigrants, the unbanked, veterans, single parents, and more. Built for the State Farm Financial Wellness track at Innovation Hack 2026.

## What It Does

1. User answers 8 intake questions about their financial situation
2. GraphRAG traverses a knowledge graph built from verified financial documents
3. A deterministic FOO (Financial Order of Operations) rule engine orders their steps by math
4. Gemini personalizes the language to be warm and specific to their situation
5. An interactive knowledge graph visualization shows the reasoning path
6. What-If scenarios let users simulate changes and see their plan reorder instantly

## Architecture

```
User fills 8-question intake form
        |
        v
query_graph.py: GraphRAG traversal
  -> maps answers to 4,194 entity knowledge graph
  -> discovers persona-specific nodes (veteran benefits, immigrant guides, etc.)
        |
        v
foo_engine.py: 10 deterministic FOO rules
  -> orders steps by financial math (28% APR before 0.5% savings, etc.)
  -> State Farm card positioned at mathematically correct step
  -> insurance filtered by user's actual situation (car, renters, dependents)
        |
        v
personalize.py: single Gemini LLM call
  -> rewrites step labels in warm, personal language
  -> echoes user's own words from Q6
        |
        v
Frontend renders:
  -> animated GraphRAG traversal (hop-by-hop path illumination)
  -> priority queue with FOO-ordered steps
  -> interactive vis.js knowledge graph
  -> What-If scenario comparison with diff badges
```

## Knowledge Graph

- **4,194 entities** extracted by Gemini 2.5 Flash via GraphRAG
- **8,066 edges** (relationships between entities)
- **21 communities** (Louvain clustering)
- **Entity types:** FINANCIAL_CONCEPT, ACTION, FINANCIAL_PRODUCT, ORGANIZATION, RISK, GEO, PERSON, EVENT
- **4,194 entity embeddings** (3072-dim) for Q6 semantic search
- **21 community reports** generated by Gemini

## Personas (Graph-Derived, Not Hardcoded)

Personas are detected by checking which corpus entity clusters the user's graph traversal touches:

| Persona | Corpus Entities |
|---------|----------------|
| International Student | INTERNATIONAL STUDENTS, NEWCOMER'S GUIDES, ITIN |
| Recent Immigrant | IMMIGRANTS, FOREIGN DRIVERS, REMITTANCE TRANSFER PROVIDERS |
| Gig Worker | IRREGULAR INCOME, SELF-EMPLOYMENT, CASH |
| Single Parent | FAMILY MEMBER, PUBLIC BENEFITS, LIHEAP |
| First-Gen Student | STUDENT LOANS, FEDERAL STUDENT LOANS, BORROWER |
| Unbanked | PREPAID CARD, MONEY ORDER, CHECK CASHING STORE |
| Veteran | VETERAN, ACTIVE DUTY SERVICEMEMBERS, MILITARY LENDING ACT |
| Reentry | FOCUS ON REENTRY COMPANION GUIDE, REENTRY |
| General | Fallback when no cluster matches |

Users can select multiple personas (e.g., veteran + single parent) — each adds its entity cluster to the graph traversal.

## FOO Engine — 10 Rules

| Rule | Logic | Exception |
|------|-------|-----------|
| 1 | No bank account = always Step 1 | None |
| 2 | High-APR debt (>15%) = pay before saving | If savings < $500, build buffer first (Rule 3) |
| 3 | Savings < $500 = $500 buffer before debt | One-incident protection minimum |
| 4 | Insurance gap = State Farm card after income stabilizes | Not before cash flow exists |
| 5 | Gig/cash income = emergency fund is $1,000 not $500 | Irregular income needs bigger buffer |
| 6 | No credit history = secured card after bank account | Prerequisite: bank account first |
| 7 | Student loans = IDR check before standard repayment | May result in $0/month payment |
| 8 | Employer 401k match = catch it before extra debt | Free money beats debt interest math |
| 9 | Multiple debts = avalanche (highest APR first) | Mathematically optimal |
| 10 | Savings > $1k + no high-APR debt = investment unlocks | Sequence gate |

**Insurance is filtered by user's actual situation:**
- Auto insurance only recommended if user has a car
- Renters insurance only if user rents
- Life insurance only if user has dependents
- Health insurance always recommended

## State Farm Integration

- Embedded as a numbered step in the priority queue (not a banner)
- Position determined by FOO math: crisis = Step 6-7, stable = Step 2-3
- Card text includes actual savings amount
- Clicking the State Farm badge links to the correct insurance page on statefarm.com
- If user already has all relevant insurance, card does not appear

## What-If Simulator

6 pre-built scenarios that re-run FOO only (~50ms, no GraphRAG re-query):

| Scenario | Profile Change |
|----------|---------------|
| What if I lose my job? | income → irregular |
| What if I earn $200 more? | income boost |
| What if I pay off my debt? | debt → none |
| What if I get renters insurance? | adds renters to Q5 |
| What if I open a bank account? | bank account → yes |
| What if I get a steady job? | income → salary |

**Diff indicators show what changed:** NEW (green), MOVED with old position (yellow), REMOVED (red strikethrough). "Back to my plan" restores original queue.

- **GraphRAG traversal animation** — hop-by-hop path illumination with:
  - Entry nodes highlighted first
  - Active nodes scale 3.2x with glow
  - Edge glow with dashed animation

- **Interactive vis.js knowledge graph** with bidirectional step-node interaction
- **Floating reasoning popover** on step click
- **What-If buttons** with diff comparison

## API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/query` | POST | Full pipeline: graph traversal → FOO → personalization |
| `/api/whatif` | POST | What-If scenario (re-runs FOO only, ~50ms) |
| `/api/search` | POST | Semantic search for Q6 free text |
| `/api/community/<id>` | GET | Community info with report |
| `/api/node/<name>` | GET | Node neighborhood (click-to-expand) |
| `/api/health` | GET | Health check |
| `/api/personas` | GET | Demo personas |
